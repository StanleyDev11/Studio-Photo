🔴 L’erreur importante est celle-ci :
java.lang.RuntimeException:
L'API Fedapay n'a pas retourné d'URL de paiement valide.


Donc :

👉 Ton backend appelle l’API FedaPay
👉 FedaPay répond
👉 MAIS dans la réponse il n’y a PAS d’URL de paiement
👉 Ton code lance donc une RuntimeException

🎯 Ce que ça veut dire concrètement

Dans ta méthode :

FedapayService.initiatePayment(...)


À la ligne :

FedapayService.java:165


Tu dois avoir quelque chose comme :

if (paymentUrl == null) {
    throw new RuntimeException("L'API Fedapay n'a pas retourné d'URL de paiement valide.");
}


Donc le problème vient de :

👉 La réponse JSON de FedaPay n’est pas celle attendue.

🔎 Les causes possibles
1️⃣ Clé API incorrecte ou absente

Vérifie que ton container a bien :

-e FEDAPAY_API_KEY=xxxx


Ou que dans ton application.properties :

fedapay.api.key=xxxx


Si la clé est mauvaise → FedaPay renvoie une erreur.

2️⃣ Mauvais environnement (sandbox vs production)

FedaPay a :

Sandbox

Live

Si tu utilises une clé sandbox avec l’URL production → ça ne marche pas.

3️⃣ FedaPay renvoie une erreur JSON

Il est possible que FedaPay renvoie :

{
  "message": "Unauthorized"
}


Mais ton code ne vérifie pas correctement la réponse.

🔥 Ce que tu dois faire maintenant

Ajoute un log pour afficher la réponse complète de FedaPay.

Dans FedapayService juste après l’appel API :

System.out.println("Fedapay response: " + responseBody);


Ou :

log.info("Fedapay raw response: {}", responseBody);
